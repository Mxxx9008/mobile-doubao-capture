# 豆包 API 抓包工具 — 初始使用说明

> 如何在全新设备上跑通这套工具链，从零到产出数据。

---

## 一、硬件/系统要求

- Windows / macOS / Linux 均可（Python 脚本跨平台）
- 一个 Android 模拟器（推荐 Mumu，ADB 开箱即用）或 Android 真机
- 能访问 GitHub（下载 frida-server 需要科学上网）

---

## 二、克隆仓库

```bash
git clone https://github.com/Mxxx9008/mxxx-shameless.git
cd mxxx-shameless
```

---

## 三、安装 Python 依赖

```bash
pip install frida mitmproxy
```

> ⚠️ **门槛 1：frida Python 包和 frida-server 版本必须严格一致**
>
> 比如 `pip install frida` 装了 17.9.8，Android 端的 frida-server 也必须是 17.9.8，高一点低一点都不行。

检查版本：
```bash
pip show frida | grep Version
# → Version: 17.9.8
```

---

## 四、配置 Android 环境

### 4.1 安装模拟器 & 豆包

- Mumu 模拟器官网下载安装
- 模拟器内应用商店搜索"豆包"安装
- 下载 [Android Platform Tools](https://developer.android.com/tools/releases/platform-tools)（只需 adb）

### 4.2 连接 ADB

```bash
adb connect 127.0.0.1:5555
adb devices
# 确认设备在线
```

> ⚠️ **门槛 2：不同模拟器 ADB 端口不同**
>
> | 模拟器 | ADB 地址 |
> |--------|-----------|
> | Mumu | 127.0.0.1:5555 |
> | 雷电 | 127.0.0.1:5555 |
> | 夜神 | 127.0.0.1:62001 |
> | AVD | 127.0.0.1:5554 |
> | 真机 (USB) | USB 调试 + OEM 驱动 |

### 4.3 安装 frida-server

这是最容易卡住的一步。

```bash
# 1. 确认 Android 架构
adb shell getprop ro.product.cpu.abi
# → arm64-v8a  /  x86_64  /  armeabi-v7a

# 2. 从 GitHub Releases 下载对应架构的 frida-server
# https://github.com/frida/frida/releases
# 文件名格式：frida-server-{版本号}-android-{架构}.xz

# 3. 解压并推入设备
xz -d frida-server-17.9.8-android-arm64.xz
adb push frida-server-17.9.8-android-arm64 /data/local/tmp/frida-server
adb shell chmod 755 /data/local/tmp/frida-server
```

> ⚠️ **门槛 3：GitHub 可访问性**
>
> 国内网络可能无法直接访问 GitHub。解决方案：
> - 科学上网后浏览器下载
> - 使用 GitHub 镜像站（如 ghproxy.com）
> - 从已有设备的 `frida-server` 文件直接拷贝

## 五、启动抓包流水线

需要同时跑 **3 个进程**，建议开 3 个终端窗口。

### 终端 1：启动 frida-server

```bash
adb root                           # 必须 root，否则无法注入
adb shell /data/local/tmp/frida-server &
```

> ⚠️ **门槛 4：adb root 支持**
>
> 不是所有模拟器都支持 `adb root`。Mumu 可以。
> 真机需要 Magisk 或类似 root 方案。

### 终端 2：启动 mitmproxy 代理

```bash
mitmdump -w /tmp/capture.mitm
```

### 终端 3：设置代理 + Frida 注入

```bash
# 模拟器走宿主机代理
adb shell settings put global http_proxy 10.0.2.2:8080

# 注入 SSL bypass 脚本到豆包进程
python frida_inject.py
```

> ⚠️ **门槛 5：模拟器宿主机地址**
>
> `10.0.2.2` 是 Mumu/标准 AVD 的宿主机地址。
> 其他模拟器可能需要 `192.168.x.x` 或桥接网卡地址。
> 可以在模拟器里 `ping` 宿主机确认。

> ⚠️ **门槛 6：Frida 600 秒超时**
>
> Frida 注入有效期约 10 分钟。
> 如果开新对话框时已超时，重新跑 `python frida_inject.py` 即可。

## 六、抓取数据

### 6.1 正常提问

三个进程都跑起来后，在模拟器的豆包 App 里正常输入问题、等待回答。

### 6.2 停止抓包

- **Ctrl+C** 停止 mitmdump
- mitmdump 会自动保存 `capture.mitm` 文件

### 6.3 导出文本格式

```bash
mitmdump -r /tmp/capture.mitm --no-http2 > capture.txt
```

### 6.4 提取参考文献

修改 `extract_references.py` 里的路径：

```python
CAPTURE = r"你的capture.txt路径"
OUTPUT = r"你想要的输出目录"
```

然后运行：

```bash
# Windows 必备环境变量，否则中文乱码
PYTHONIOENCODING=utf-8 python extract_references.py
```

### 6.5 查看结果

输出文件：`doubao_conv_{会话ID}.json`，每个对话框一个文件。


## 七、恢复代理设置

用完记得恢复模拟器的代理，否则关掉 mitmproxy 后模拟器无法上网：

```bash
adb shell settings put global http_proxy :0
```

## 九、门槛清单（按坑人程度排序）

| 排名 | 门槛 | 卡住概率 | 解决 |
|------|------|------|------|
| 🥇 | frida-server 版本不匹配 | 高 | `pip show frida` 看版本，下完全一致的 server |
| 🥈 | GitHub 不可达 | 高 | 科学上网 / 镜像站 |
| 🥉 | `adb root` 不支持 | 中 | 换 Mumu 模拟器，或真机 Magisk |
| 4 | 模拟器代理地址不对 | 中 | 确认宿主机 IP，Mumu 用 `10.0.2.2` |
| 5 | Frida 600s 超时 | 中 | 新对话前重新注入 |
| 6 | Windows GBK 编码 | 低 | 命令加 `PYTHONIOENCODING=utf-8` |
| 7 | Python 版本太旧 | 低 | 3.10+ 即可 |
