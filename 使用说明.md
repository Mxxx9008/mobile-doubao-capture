# mobile-doubao-capture 使用说明

> 豆包 App 参考文献抓取工具 — 从零部署到日常使用，一份文档全覆盖

---

## 一、项目简介

跟豆包聊天时，它会引用参考文献（网页链接、来源摘要等）。这套工具自动拦截豆包的 HTTPS 流量，提取每条回答背后的参考文献，按对话框分组保存为结构化 JSON 文件。

**核心能力：**
- 绕过豆包 SSL Pinning，解密 HTTPS 明文
- 提取参考文献（标题、URL、来源、摘要）
- 提取搜索关键词和回答全文
- 按对话框自动分组，同一对话框多次提问自动追加
- 品牌识别 + 来源统计

## 三、首次配置（新设备只需做一次）

### 3.1 克隆项目

```bash
git clone https://github.com/Mxxx9008/mobile-doubao-capture.git
cd mobile-doubao-capture
```

### 3.2 安装 Python 依赖

```bash
pip install frida mitmproxy
```

> ⚠️ **frida Python 包和 frida-server 版本必须严格一致**，高一点低一点都不行。

确认版本：
```bash
pip show frida | grep Version
# → Version: 17.9.8
```

### 3.3 安装模拟器 & 豆包

- Mumu 模拟器官网下载安装
- 模拟器内应用商店搜索"豆包"安装

### 3.4 连接 ADB

```bash
adb connect 127.0.0.1:5555
adb devices
```

> 不同模拟器的 ADB 端口：

| 模拟器 | ADB 地址 |
|--------|-----------|
| Mumu | 127.0.0.1:5555 |
| 雷电 | 127.0.0.1:5555 |
| 夜神 | 127.0.0.1:62001 |
| AVD | 127.0.0.1:5554 |
| 真机 (USB) | USB 调试 + OEM 驱动 |

### 3.5 安装 frida-server

这是最容易卡住的一步。

```bash
# 1. 确认 Android 架构
adb shell getprop ro.product.cpu.abi
# → arm64-v8a  /  x86_64  /  armeabi-v7a

# 2. 下载对应版本（版本号必须和 pip 的一致）
# https://github.com/frida/frida/releases
# 文件: frida-server-17.9.8-android-arm64.xz

# 3. 解压并推入
xz -d frida-server-17.9.8-android-arm64.xz
adb push frida-server-17.9.8-android-arm64 /data/local/tmp/frida-server
adb shell chmod 755 /data/local/tmp/frida-server
```

> GitHub 不可达？科学上网 / 镜像站（ghproxy.com）/ 从旧设备拷贝。

## 四、一键启动（日常使用）

首次配置完成后，日常抓包只需这一条命令。

### 4.1 准备工作

1. 启动 Mumu 模拟器
2. 打开豆包 App（确保在主界面）

### 4.2 运行脚本

```bash
bash start_capture.sh
```

### 4.3 看到就绪提示

```
============================================
   ALL SET - Go chat in Doubao!
============================================

[*] Capture file: /c/Users/.../Desktop/doubao_capture_20260513_143025.mitm
Press Enter when you're done chatting to extract data...
```

### 4.4 提问 & 提取

1. 在豆包里正常输入问题，等待回答完成
2. 可以连续问多个问题
3. 问完后，回到终端按 **Enter**
4. 脚本自动：停止抓包 → 解析数据 → 生成 JSON → 放到桌面

### 4.5 输出

桌面得到 `doubao_conv_{会话ID}.json`，每个对话框一个文件。

## 六、手动操作（分步调试用）

一键脚本封装了下面的全部步骤。如果出问题，可以手动执行排查：

```bash
# ===== 终端 1：frida-server =====
adb connect 127.0.0.1:5555
adb root
adb shell /data/local/tmp/frida-server &

# ===== 终端 2：mitmproxy =====
mitmdump -w capture.mitm

# ===== 终端 3：代理 + 注入 =====
adb shell settings put global http_proxy 10.0.2.2:8080
python frida_keepalive.py

# ===== 在豆包提问 =====

# ===== 提取数据 =====
mitmdump -r capture.mitm --no-http2 > capture.txt
PYTHONIOENCODING=utf-8 python extract_references.py capture.txt

# ===== 用完恢复代理 =====
adb shell settings put global http_proxy :0
```

> **关于代理地址**：`10.0.2.2` 是 Mumu/标准 AVD 的宿主机地址，其他模拟器可能不同。在模拟器里 ping 宿主机确认。

## 八、常见问题

### Q: 脚本报 "ADB connection failed"

模拟器没开，或 ADB 端口变了。
```bash
adb connect 127.0.0.1:5555
adb devices  # 确认设备在线
```

### Q: 脚本报 "frida-server failed to start"

frida-server 没 push 或 `adb root` 失败。
```bash
adb root
adb push frida-server /data/local/tmp/frida-server
adb shell chmod 755 /data/local/tmp/frida-server
```

### Q: 豆包回答里面有参考文献，但 JSON 里是空的

Frida 注入超时了，重新注入后再提问：
```bash
python frida_keepalive.py &
```

### Q: 中文在终端显示乱码

所有 Python 命令前加 `PYTHONIOENCODING=utf-8`。一键脚本已经自动处理了。

### Q: 模拟器关了之后网络连不上

代理没恢复。手动恢复：
```bash
adb shell settings put global http_proxy :0
```
